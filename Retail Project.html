Q3

retail %>%
  features(Turnover, features = guerrero) %>% 
  select(lambda_guerrero)
# A tibble: 1 × 1
  lambda_guerrero
            <dbl>
1          0.0172

This means that for the given Turnover time series, the Guerrero method suggests a Box-Cox transformation parameter (λ lambda) of approximately 0.017.

With the λ lambda parameter being close to 0, we should use a log transformation since it should be most appropriate for variance stabilisation.

Now we will apply Box-Cox transformation using the given lambda value.

retail_bc <- retail %>%
  mutate(BoxCox_Turnover = box_cox(Turnover, lambda = 0.017))

retail_bc %>%
  select(Month, Turnover, BoxCox_Turnover) %>%
  pivot_longer(cols = c(Turnover, BoxCox_Turnover)) %>%
  autoplot(value) +
  facet_wrap(~name, scales = "free_y") +
  labs(title = "Box-Cox Transformed Turnover vs Original")

The plot has a noticeably more stable variance over time compared to the original data. This confirms that the Box-Cox transformation is effective in stabilising the variance, thus addressing heteroskedasticity.

dcomp <- retail_bc %>% 
  model(stl = STL(BoxCox_Turnover ~ trend(window = 13) + season(window = "periodic"), robust = TRUE))
components(dcomp) %>% autoplot + labs(title = "STL decompisition: Turnover")

The top panel of the STL decompisition shows a general upward trend and potential seasonality

The second panel shows there are periods with slight dips, but there is general upward movement in the trend with slight variation

Setting trend(window = 13) smooths over roughly a year. A 13-month window for the trend component used to smooth short-term fluctuations while preserving the long-term trend.

The third panel shows consistent seasonality, steadily increasing until roughly April where it dips slightly, to be followed up with another upward trend until roughly October, where it drastically decreases

With ‘robust = TRUE’, we have ensured that the occasional unusual observations will not affect the estimates of the trend-cycle and seasonal components.They will, however, affect the remainder component.

The fourth panel appears that the remainder component could be white noise, with a few spikes.

residuals_tsibble <- components(dcomp) %>%
  select(remainder) %>%
  as_tsibble()

residuals_acf <- residuals_tsibble %>% 
  ACF(remainder)

residuals_acf %>% 
  autoplot() + 
  labs(title = "White Noise Test")

Since some of the autocorrelation coefficients lie outside these limits, we can confirm that the data is not white noise.

Q6 and 7

To create a short-list of appropriate ARIMA and ETS models, we will follow the following methodology :

  • Step 1: The full dataset will be split into a training set and a test set. The training set will include all observations up to December 2020, and the test set will consist of the final 24 months of data: January 2021 to December 2022.

  • Step 2: Rather than manually testing many different combinations, automated functions ARIMA() and ETS() were used to identify optimal models based on AIC.

  • Step 3: For the models residual diagnostics is conducted using ljung-Box tests to ensure no significant autocorrelation remains in residuals. In addition time plots and ACF plots of residuals are also used to check for randomness and constant variance.

retail_train <- retail_bc %>% filter(Month < yearmonth("2021 Jan"))
retail_test <- retail_bc %>% filter(Month >= yearmonth("2021 Jan"))

Q11

ARIMA Model

Benefits:

  • Captures complex non-seasonal and seasonal temporal dependencies.
  • Often more accurate when seasonal patterns are stable but not perfectly regular.
  • Can handle cycles.
  • Can handle dynamic regression

Limitations:

  • Needs the data to be stationary or transformed accordingly (e.g., log, difference).
  • Can struggle with irregular seasonality or sudden changes (e.g., Covids impact).
  • Can struggle with forecasting

ETS Model

Benefits:

  • Automatically captures trend and seasonality, often more robust to data irregularities.
  • Good for short-term forecasting when patterns are smooth and regular.
  • Easier to interpret and tune.

Limitations:

  • Less flexible than ARIMA if data has irregular shocks or multiple seasonalities.
  • May overfit if the data contains outliers or structural changes.
  • Structural changes (eg. GFC and COVID) are not explicitly modeled. Sudden shifts in consumer behaviour (eg. saving money and lockdowns) may reduce model reliability.
  • Assumes seasonal pattern stability: ETS assumes constant seasonal patterns, but retail patterns may evolve (e.g. online shopping growth).
  • Unlike ARIMA, cannot handle cycles
  • Unlike ARIMA, cannot handle dynamic regression
  • Like ARIMA can struggle with forecasting
